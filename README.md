<h2><strong>  :white_medium_square:  Algoritmo Busca de Usuário :white_medium_square:  User Search Algorithm </strong></h2>

<p align="justify">:pencil: Nesta etapa, você trabalhará com a implementação das estruturas de controle, repetição, estruturas homogêneas e função. O sistema deverá realizar o cadastro dos usuários em uma estrutura homogênea e atender um dos requisitos do cliente que solicita a funcionalidade que permite realizar a busca de um usuário pelo seu nome. Para tanto, deverá ser implementado um algoritmo em pseudocódigo, com as seguintes especificações: 

Implementação de um laço de repetição que solicite as informações do usuário até que seja informado o número 4 para sair do sistema. A cada repetição, apresentar o menu com as seguintes opções:   </p>

1 – Cadastrar novo usuário: solicita as informações do usuário, que são Nome e Idade.;  

2 – Listar todos os usuários cadastrados;  

3 – Buscar usuário pelo nome; 

4 – Sair do sistema: encerra o comando de repetição e encerra o algoritmo.  

<p align="justify">Para a implementação da funcionalidade de busca de um usuário pelo nome, é necessário implementar uma função que receba por parâmetro o nome de uma pessoa. Esta função verificará todas as posições da estrutura homogênea até encontrar uma que coincida com o nome passado por parâmetro ou finalizar a pesquisa sem encontrar a pessoa. Ao encontrar, deverá apresentar o nome e a idade da pessoa e retornar a sua posição, caso não encontre deverá retornar -1.   </p>

Após selecionar esta opção, deverá ser solicitado o nome do usuário, para caso encontrado, efetivar a sua remoção da estrutura homogênea. </p>

<h2><strong>  :white_medium_square:  Algoritmo Implementação de Pilha :white_medium_square:  Stack Implementation Algorithm </strong></h2>

<p align="justify">:pencil: A empresa R2D2 foi contratada para dar continuidade em funcionalidades específicas de software de cadastro de clientes e uma das demandas é que os usuários deverão ser incluídos em ordem de Pilha no sistema, pois este formato atende o modelo de negócio do cliente.

Sabendo da importância das estruturas de dados e como manipulá-las, você deverá implementar no VISUALG a estrutura de Pilha, lembrando que este tipo de estrutura segue o conceito de que o primeiro a entrar será o último a sair (FILO – First in Last Out). A Pilha deverá ser implementada utilizando uma estrutura homogênea com 20 posições e as seguintes funções deverão ser implementadas para a manipulação da pilha: 

 Empilhar (adiciona um elemento no topo da pilha); 

 Desempilhar (remove e retorna um elemento do topo da pilha); 

 Limpar (remove todos os elementos da pilha); 

 Listar (lista todos os elementos armazenados na pilha); 

 Vazia (retorna verdadeiro se a pilha estiver vazia, e falso caso contrário). 
 
Obs. 1: Lembre-se de realizar todos os tratamentos necessários para que não aconteça o estouro da estrutura homogênea.  </p>

<h2><strong>  :white_medium_square:  Algoritmo Ordenação Bubblesort :white_medium_square:  Bubblesort Sorting Algorithm </strong></h2>

<p align="justify">:pencil: A empresa R2D2 precisa resolver a seguinte situação: o cliente possui um sistema de cadastro de senhas de chamados dos usuários, as senhas numéricas são entre 0001 até 1024, sendo armazenados em uma estrutura homogênea (vetor) na ordem de Pilha ou Fila. As buscas dentro do sistema de cadastros de senhas são frequentes e quando atinge um número grande de usuários cadastrados, o sistema fica lento e às vezes trava. 

Nesse momento, você deve continuar a buscar formas de otimizar as consultas das senhas dos usuários independentemente da quantidade de usuários cadastrados, para tanto é preciso estudar técnicas de ordenação e um dos primeiros testes será com o algoritmo BubbleSort (ou método Bolha). 

 

Utilizando o algoritmo Bubblesort, implemente uma função que receba por parâmetro uma estrutura homogênea desordenada com os números das senhas e aplique o algoritmo de ordenação retornando uma estrutura homogênea ordenada de forma crescente. Ao final, informe o maior e o menor número. </p>
